  2. What is Git?

    Git is a distributed version control system, meaning that it allows developers to work on their own local copies of a project, while still enabling them to push changes to a shared repository.It was created by Linus Torvalds in 2005, and has been maintained by Junio Hamano since then.

    What Is Git Version Control?

    Git as a version control system (VCS), it helps developers track changes, manage multiple versions of code, and collaborate efficiently. It is widely used in software development to ensure smooth teamwork and maintain a structured history of modifications.

    What is Version Control?

    Version control is a system that tracks changes made to files over time. It allows developers to:

    • Save and track changes: Every modification made to the codebase is recorded.
    • Revert to previous versions: If something breaks or a feature doesn't work as expected, you can revert to a stable version.
    • Collaborate: Multiple developers can work on the same project without overwriting each other's work.
    • Branching and Merging: Developers can create branches for different features, work on them independently, and merge them back to the main codebase when ready.

    Other Version Control Systems:

    1. Mercurial (Hg) - A distributed version control system similar to Git, known for its simplicity and efficiency.
    2. Subversion (SVN) - A centralized version control system used in enterprise environments.
    3. Perforce - A high-performance version control system often used in game development and large-scale projects.
    4. Bazaar - A flexible version control system that supports both distributed and centralized workflows.

      5. In this blog, I'll be focusing primarily on Git and GitHub, which are among the most popular tools for version control and collaboration. While other version control systems like Mercurial and Subversion exist, Git stands out due to its wide adoption and integration with platforms like GitHub.

    To start tracking files, you need to stage them. This moves the files into a "staging area" before committing them:

    Here are some key commands for staging:

    • git add (file) -> Stage a file
    • git add --all -> Stage all changes
    • git add -A -> Stage all changes
    • git add . -> Stage all changes
    • Git status -> See what is staged
    • git rm --cached (file) -> Unstage a file

  14. Next steps

    In this blog we have only the basic of Git. You should find extra resources and tutorials to advance your skills. Here are some of the key area you'd like to look at:

    Git push operation

    The git push command is used to push the commits from our local branch to the relevant branch on the remote repository.

    git push -u (remoteName) (branchName)

    Git clone operation

    This command is used to clone a remote repository into our local machines.

    git clone (remote url)

    Git Tagging Operation

    Git tags are used to highlight significant events in a project's history. They are usually associated with releases (like v1.0.2, v2.1.0) or milestones in the development of a project.

    • Lightweight tags: Eg. git tag (tagName)
    • Annotated tags: Eg. git tag -a (tag-name) -m "Tag message"
